Java的线程
线程 Java
2018-10-21

这一篇博客将会用来写Java中从浅至深线程相关的东西,现在更新中。

线程的创建

除了众所周知的直接继承于Thread和实现Runnable的方法以外，还有Java 5后提供的Future+Callable，先分别写写他们创建线程的方式与区别。

Thread

直接通过继承于Thread的方式来创建：

  1. 需要一个继承于Thread的类，并重写其中的run()方法
  2. 新建该类的对象

  3. 通过该对象的start()启动线程

    public class TestThread extends Thread{
    private int i ;
    @Override
    public void run() {
        for( ; i <100 ; i++){
            System.out.println(getName()+" "+i);
        }
    }
    public static void main(String[] args) {
        new TestThread().start();
        new TestThread().start();
        for(int i = 0 ; i < 100 ; i ++){
            System.out.println(Thread.currentThread().getName()+" "+i);
        }
    }
}

此时输出的效果部分如下：

main 69
Thread-1 42
main 70
Thread-0 66
Thread-0 67
main 71
Thread-1 43
Thread-1 44
main 72
Thread-0 68
main 73
Thread-1 45
main 74
Thread-0 69

可以看见，三个线程都混杂在一起，谁也不能影响到谁的变量i，这是以继承的方式。

Runnable

Runnable的创建线程的方式：

  1. 需要一个实现Runnable的类，并实现其中的run()方法
  2. 创建这个类的实例，作为参数创建一个新的Thread对象

  3. 通过该对象的start()启动线程

    public class TestRunnable implements Runnable{

    private int i;

    public void run() {
        for (; i < 100; i++) {
            System.out.println(Thread.currentThread().getName() + " " + i);
        }
    }
    public static void main(String[] args) {
        TestRunnable tr = new TestRunnable();
        new Thread(tr,"Runnable-Thread1").start();
        new Thread(tr,"Runnable-Thread2").start();
        for (int i = 0; i < 100; i++) {
            System.out.println(Thread.currentThread().getName() + " " + i);
        }
    }
}

    此时输出的效果部分如下：

    Runnable-Thread1 33
    main 50
    Runnable-Thread2 34
    main 51
    Runnable-Thread1 35
    main 52
    main 53
    Runnable-Thread2 36
    Runnable-Thread2 38
    Runnable-Thread2 39
    Runnable-Thread2 40
    Runnable-Thread2 41
    main 54
    Runnable-Thread1 37

虽然主线程和我们创建的线程的变量还是互不干扰，但Thread1和Thread2明显共用了同一个属性，可见，当两个两个线程同时以同一个Runnable的实现类作为参数的时候，他们能共用那个runnable的属性。（当然要是你创建两个Runnable实例效果会跟Thread一样，但是那样就没用Runnable的意义了）
但因为getName()是Thread类里的方法，所以必须用Thread.currentThread()先获得当前线程对象才能调用getName()。

Callable

Callable创建线程的方式跟Runnable很类似：

  1. 需要一个实现Callable的类，并实现其中的call()方法
  2. 创建这个类的实例，作为参数包装成一个新的FutureTask对象
  3. FutureTask作为参数创建一个新的Thread对象
  4. 通过该对象的start()启动线程
    public class TestCallable implements Callable<Integer> {

        private int i;

        @Override
        public Integer call(){
            for(; i < 100 ; i++){
                System.out.println(Thread.currentThread().getName()+" "+i);
            }
            return i;
        }

        public static void main(String[] args) throws ExecutionException, InterruptedException {
            TestCallable tc = new TestCallable();
            FutureTask<Integer> task = new FutureTask<>(tc);
            new Thread(task,"Callable-Thread").start();
            for(int i = 0 ; i <100 ; i++){
                System.out.println(Thread.currentThread().getName()+" "+i);
            }
            try {
                System.out.println("返回值"+task.get());
            } catch (Exception e) {
                e.printStackTrace();
            } 
        }
    }

部分输出效果：

main 92
main 93
main 94
Callable-Thread 95
Callable-Thread 96
main 95
main 96
main 97
Callable-Thread 97
Callable-Thread 98
main 98
Callable-Thread 99
main 99
返回值100

虽然用Callable创建起来比Runnable要复杂，但从这里可以看出来了：利用Callable不仅可以抛出异常，而且可以通过get()方法获得线程执行返回的信息。

其次要注意的是，这里我没有创建两个线程，只有一个主线程和一个Callable的线程，是因为使用同一个FutureTask来创建线程会被认为是“不安全的”，在Callable的线程体call()执行一次之后会将status改成不再是NEW，而不是NEW的callable将直接退出，无法执行线程体：

public void run() {
    if (state != NEW ||
        !UNSAFE.compareAndSwapObject(this, runnerOffset,null, Thread.currentThread()))
    return;

所以如果用像前面Runnable那样用一个target创建两个线程的话，第二个线程将会在执行线程体之前就挂掉。

三者比较

用实现Runnable的方法创建多线程的优势：

1.能继承与实现更多的类
2.能通过共用target来进行贡献资源

用实现Callable的方法创建多线程的优势：

1.能继承与实现更多的类
2.能捕获异常进行处理
3.能获得线程执行的返回值，在编程上能有更多空间

用Thread的方法创建多线程：

编程简单，像获得当前线程直接通过实例的方法即可。
